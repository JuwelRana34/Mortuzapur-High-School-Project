```bash

```

```bash

```

### একবারের জন্য টুল চালানো

```bash
pnpm dlx <tool>
```

## ⚠️ Build Approval System (গুরুত্বপূর্ণ)

pnpm কিছু প্যাকেজকে সরাসরি install হওয়ার পরেও block করতে পারে যদি তারা "build script" বা "native code" রান করতে চায়। এটা security feature।

### কেন এটা হয়?

কিছু প্যাকেজ install হওয়ার সময় নিচের কাজগুলো করে:

* বাইনারি ফাইল ডাউনলোড করে
* C/C++ native code compile করে
* runtime tool (যেমন Cloudflare worker runtime) সেটআপ করে

pnpm এগুলো auto-run হতে দেয় না যতক্ষণ না তুমি manually approve করো।

---

### 🚨 কখন এই error আসে?

এই টাইপ message দেখলে বুঝবে:

```
[ERR_PNPM_IGNORED_BUILDS]
Ignored build scripts
```

মানে কিছু package install হয়েছে, কিন্তু তার build step block করা হয়েছে।

---

### ✅ কীভাবে ঠিক করবে (Proper Flow)

#### Step 1: approval tool চালাও

```bash
pnpm approve-builds
```

#### Step 2: list আসবে → বুঝে select করো

Common safe choices:

* esbuild → build tool (Next.js / bundler)
* sharp → image optimization
* workerd → Cloudflare runtime
* msw → API mocking (dev only)
* unrs-resolver → dependency resolver

#### Step 3: select করার নিয়ম

* SPACE → select/unselect
* ENTER → confirm list
* Y → final approve

---

### 🔁 তারপর কী করবে?

Approve করার পর অবশ্যই:

```bash
pnpm install
```

যদি CLI আবার আগের কমান্ড fail দেখায়:

* একই install command আবার চালাও (2nd run usually fixes it)

---

### 🧠 মনে রাখার সহজ নিয়ম

* Pure JS library → approve লাগে না
* Native / runtime / build tool → approve লাগে
* একবার approve করলে সাধারণত আবার লাগে না (same version থাকলে)

# Frontend Architecture & Folder Structure

This project follows a highly scalable **Feature-Sliced Design (FSD)** architecture, optimized for Next.js 16, TypeScript, Tailwind CSS, Shadcn UI, and Hono API integration. 

This structure ensures clean separation of concerns, high maintainability, and prevents "spaghetti code" as the application scales.

## 📂 Directory Tree

```text
src/
├── app/                    # 1. Routing Layer (Next.js App Router)
│   ├── (auth)/             # Route Groups (e.g., login, register)
│   ├── (main)/             # Main application routes
│   ├── layout.tsx          # Root Layout & Font Definitions
│   └── globals.css         # Global Styles & Tailwind Directives
│
├── components/             # 2. Shared UI Layer
│   ├── ui/                 # Dumb components generated by Shadcn UI (button, card)
│   ├── shared/             # Custom reusable components (Navbar, Footer, Modal)
│   └── providers/          # Global Context Providers (ThemeProvider, Query)
│
├── features/               # 3. Core Logic Layer (Feature Modules)
│   ├── events/             # 📦 Domain: Events
│   │   ├── actions.ts      # Server Actions (Zod Validation & Next.js Mutations)
│   │   ├── components/     # Feature-specific UI (RegistrationForm, EventCard)
│   │   ├── hooks/          # Client-side state & UI logic
│   │   ├── schemas.ts      # Zod/valibot validation schemas
│   │   ├── services.ts     # Data fetching & API communication (Hono RPC)
│   │   └── types.ts        # TypeScript Interfaces
│   │
│   └── auth/               # 📦 Domain: Authentication
│
├── lib/                    # 4. Utilities & Configurations
│   ├── api-client.ts       # Hono RPC setup (`hc`) / express API Fetcher
│   ├── utils.ts            # Helper functions (e.g., Tailwind `cn()`)
│   └── constants.ts        # Static global data (NavLinks, SiteConfig)
│
├── hooks/                  # 5. Global Custom Hooks (useMediaQuery, useDebounce)
├── stores/                 # 6. Global State Management (Zustand/Redux)
└── types/                  # 7. Global TypeScript Declarations
---
